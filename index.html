文件上传漏洞

文件上传漏洞原理

文件上传漏洞就是利用服务端代码对文件上传路径变量过滤不严格将可执行的文件上传到一个到服务器中 ，再通过URL去访问以执行恶意代码

绕过方法

1.在前端禁用js 可以使用浏览器插件在页面上禁用掉js就能成功上传

2.通过抓包修改数据包 抓包后更改Content-Type为允许的类型绕过该代码限制，比如将php文件的Content-Type:application/octet-stream修改为image/jpeg、image/png、image/gif 通过修改数据包以后就能成功上传

3.大小写绕过比如说 .pHP .aSP

4.双写绕过 pphphp

5.00截断 使用%00对文件名进行截断 1.php%00.jpg 当服务器检测到%00后就结束了

6.修改.user.ini文件

7.修改.htaccess文件

8.条件竞争通过burp抓包上传一个写马的马的文件同时访问和上传一直执行直到有burp有一条访问成功为止

跨站脚本漏洞（xss）

xss的基本概念

  1. xss漏洞一直被评估为web漏洞中危害较大的那种，在OWASP TOP10中一直处于前三的江湖地位
  2. xss漏洞是一种发生在前端的漏洞，所以他的危害对象主要也是前端用户。
  3. xss漏洞可以用来进行钓鱼攻击、前端js挖矿、用户cookie的获取。甚至可以结合浏览器自身的漏洞对用户主机进行远程控制等等。

xss的攻击流程

xss漏洞的常见类型

反射型
存储型（持久型xss）
dom型

xss漏洞形成的原因

形成xss漏洞的主要原因是程序对输入和输出的控制不够严格，导致一些精心构造的脚本输入后，在输入到前端时被浏览器当作有效代码解析执行从而产生危害。

xss漏洞测试流程

  1. 在目标站点上找到输入点，比如说查询接口留言板等等；
  2. 输入一组“特殊字符+唯一识别字符”，点击提交后，查看返回的源码，是否有做对应的处理
  3. 通过搜索定位到唯一字符，结合唯一字符前后语法确认是否可以构造执行js的条件( 构造闭合 )
  4. 提交构造的脚本代码(以及各种绕过姿势)，看是否可以成功执行，如果成功执行则说明存在XSS漏洞

xss攻击

首先先开启python的服务 python -m http.server 1234

数据库

什么叫数据库

数据库是以电子方式存储的系统数据集合。它可以包含任何类型的数据，包括文字、数字、图像、视频和文件。您可以使用称为数据库管理系统 (DBMS) 的软件来存储、检索和编辑数据。在计算机系统中，“数据库” 一词也可以指任何 DBMS、数据库系统或与数据库关联的应用程序。

数据库类型

关系型数据库

关系型数据库是一个结构化的数据库，创建在关系模型（二维表格模型）基础上，一般面
向于记录。
SQL 语句（标准数据查询语言）就是一种基于关系型数据库的语言，用于执行对关系型数
据库中数据的检索和操作。
主流的关系型数据库包括 Oracle、MySQL、SQL Server、Microsoft Access、DB2 等。

非关系型数据库

NoSQL(NoSQL = Not Only SQL )，意思是“不仅仅是 SQL”，是非关系型数据库的总称。
除了主流的关系型数据库外的数据库，都认为是非关系型。
主流的 NoSQL 数据库有 Redis、MongBD、Hbase、CouhDB 等

sql注入

SQL注入用户通过浏览器或者其他客户端将恶意SQL语句插入到网站参数中，网站应用程序未经过过滤，将恶意SQL语句带入数据库进行执行，通过数据库获取了敏感的信息或者执行了其他恶意操作。

sql注的原因

用户提交的参数数据未做充分检查过滤

sql注入危害

1.数据泄露：数据库中存放的用户隐私信息的泄露
2.网页纂改：通过操作数据库对特定的网页进行纂改
3.网站挂马：修改数据库一些字段的值，嵌入木马链接，进行挂马攻击
4.服务器被远程控制被安装后门：经由数据库服务器提供的操作系统支持，让黑客得以修改或控制操作系统
5.数据破坏：数据库服务器被攻击，数据库的系统管理员帐户被窜改，系统瘫痪

sql注入类型

按数据类型分可以分为数字型和字符型

如何判断是数字型还是字符型 写入1 and 1=1 再写入and 1=2 在语句最后使用–+把后面的语句进行注释 来查看页面是否正常回显来判断是字符注入还是数字注入
一般的过程为
1.判断注入点

2.猜字段数：order by

3.找回显：union select
4.然后在回显处注入 database()、user（）等等函数

按照返回结果可以分为盲注型和报错型

bool注入：bool注入是盲注的一种，与报错注入不同，bool注入没有任何报错信息输出，页面返回只有正常和不正常两种状态，攻击者只能通过返回的这两个状态来判断输入的测试SQL语句是否正确，从而判断数据库中的存储了哪些信息。

宽字节注入（）

宽字节注入漏洞产生的原因：开发者为了防止出现SQL注入攻击，将用户输入的数据用addslashes等函数进行过滤。addslashes等函数会默认将单引号等字符进行转义，这样就可以避免注入。宽字节注入产生的原因是MySQL在使用GBK编码的时候，如果第一个字符的ASCII码大于128时，会认为两个字符是一个汉字，会将后面的转义字符“\”吃掉，拼接为汉字，这样就可以将SQL语句闭合，造成宽字节注入。

绕过cdn

为什么需要DNS把域名 解析为地址？

网络通讯大部分是基于TCP/IP的，而TCP/IP是基于IP地址的，所以计算机在网络上进行通讯时只能识别如IP地址，而不能认识域名。我们没有办法记住很多的IP地址，所以我们在访问网站的时候就会输入诸如 www.baidu.com 这样的域名就能访问到相对应的网站，这些域名是有一个名为“DNS”的服务器把输入的域名解析成一个IP地址然后访问这个地址进入对应的页面。

dns

DNS(Domain Name System)翻译过来就是”域名系统“是互联网的一项服务。它作为将域名和IP地址)相互映射的一个分布式数据库，能够使人更方便地访问互联网。DNS使用TCP和UDP端口53。

CDN

cdn(Content Delivery Network或Content Distribution Network)的缩写翻译过来即内容分发网络是指一种通过互联网互相连接的电脑网络系统，利用最靠近每位用户的服务器，更快、更可靠地将音乐、图片、视频、应用程序及其他文件发送给用户，来提供高性能、可扩展性及低成本的网络内容传递给用户。
